class dstk.processing.TweetCleaner.TweetCleaner(all_ops=None, user_settings={})[source]¶

Bases: object

Processes tweet text using a set of specified operations.

NOTE: All ops is applied before user_settings, so
it is possible to combine the two.
Ex: Passing (all_ops=False, user_settings={‘lowercase’: True})
will apply only the lowercase op.
Parameters:
  • all_ops (bool) – If this is passed, the default settings will be overridden with the value passed. Ex. If True is passed all ops will be set to True.
  • user_settings ({str: bool}) – Keys are cleaning operations, values are bools: Ops which are ‘True’ will be applied to each tweet. Ops which are ‘False’ willl not.

dstk.processing.TweetExporter module¶

class dstk.processing.TweetExporter.TweetExporter(path, export_cols, aux_cols={}, order_override=None)[source]¶

Bases: object

Handles the writing of tweets to csv rows on a tweet-by-tweet basis.

Usage:
  1. Create a TweetExporter instance, initializing it with your

    desired path and export settings.

  2. For each tweet you want to export: call the instance’s

    export_tweet() method, passing the tweet object as the argument.

Parameters:
  • path (str) – The destination file-path to write output.
  • export_cols ([str]) – The columns to include in output.
  • aux_cols ({str: str}) –

    Keys are extra column names, values are the value to write in that column.

    Ex. {‘rumor’: ‘sunil’} will include an extra column called ‘rumor’ with a value of ‘sunil’.

  • order_override ([str]) – An ordered list of the export columns which will specif the order in which the columns are written to the file.
